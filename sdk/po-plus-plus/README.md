# PO++ — Protocol Orzatty SDK for C++

**PO++** is the official C++ SDK for Protocol Orzatty, the high-performance P2P E2EE networking protocol built on QUIC.

## Requirements

- A C++17 or later compiler (MSVC, GCC, Clang)
- The PO native library (`po_c.dll` / `libpo_c.so` / `libpo_c.a`)
- The `po.h` header (auto-generated by `cbindgen`)

## Installation

1. Download the latest release from [GitHub Releases](https://github.com/orzattyholding/po/releases)
2. Copy `po.h` and the library (`po_c.dll` / `libpo_c.so`) to your project
3. Link against the library in your build system

### CMake Example

```cmake
cmake_minimum_required(VERSION 3.16)
project(my_po_app LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)

add_executable(my_app main.cpp)

# Point to PO SDK
target_include_directories(my_app PRIVATE ${CMAKE_SOURCE_DIR}/po-sdk)
target_link_libraries(my_app PRIVATE ${CMAKE_SOURCE_DIR}/po-sdk/po_c.lib)
```

### MSVC (Visual Studio)

```
cl /EHsc /std:c++17 main.cpp /I. po_c.lib
```

## Quick Start

```cpp
#include <iostream>
#include <cstring>
#include "po.h"

int main() {
    // Connect to a PO server
    auto* client = po_client_new("0", "127.0.0.1:9091");
    if (!client) {
        std::cerr << "E2EE handshake failed" << std::endl;
        return 1;
    }

    // Send encrypted data over QUIC
    const char* msg = "Hello from PO++!";
    int result = po_client_send(client, 
        reinterpret_cast<const uint8_t*>(msg), 
        std::strlen(msg));

    if (result == 0) {
        std::cout << "Encrypted payload sent via QUIC/E2EE" << std::endl;
    }

    po_client_free(client);
    return 0;
}
```

## API Reference

| Function | Description |
|---|---|
| `po_client_new(bind, remote)` | Create a connection. Pass `NULL` for `remote` to run as server. |
| `po_client_send(client, data, len)` | Send encrypted data. Returns `0` on success, `-1` on error. |
| `po_client_free(client)` | Release all resources. |

## Architecture

PO++ links directly against the Rust-compiled native library via C ABI (`extern "C"`). Zero runtime overhead, zero garbage collection, zero JIT — **pure native speed with E2EE guarantees**.
